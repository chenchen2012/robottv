# Session Memory

Use this folder for short repo-tracked session notes.

## Goal
- Keep implementation context, decisions, caveats, and next steps in Git.
- Make it easy for another computer or collaborator to recover recent work with `git pull`.

## File naming
- Use one file per meaningful work session.
- Preferred format:
  - `YYYY-MM-DD-short-topic.md`

Example:
- `2026-03-08-homepage-and-live-page.md`

## Recommended template
- Date
- Scope
- Problem
- Changes made
- Files touched
- Commits
- Caveats
- Recommended next steps

## Guidance
- Keep notes concise and factual.
- Record decisions, not raw chat logs.
- Include commit hashes when code was pushed.
- If a note supersedes a root-level ad hoc file, keep only the version in this folder.
